# Security Policy

## Supported Versions

Only the latest production deployment of Slurp receives security fixes. There are no versioned releases to maintain.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email [slurp-support@mklau.net](mailto:slurp-support@mklau.net) with:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

## Disclosure Policy

- Please give us reasonable time to investigate and fix before public disclosure
- We will notify you when the fix is deployed
- We are happy to credit you in the fix announcement if you wish
